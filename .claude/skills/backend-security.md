## Authentication Middleware

### Middleware Chain Pattern

```
Request --> Rate Limiter --> Auth Middleware --> Authorization --> Route Handler
```

### Implementation (Express/Node.js)

```javascript
// Auth middleware — verify session or token before route logic
function authenticate(req, res, next) {
  const session = req.cookies?.session_id;
  if (!session) return res.status(401).json({ error: 'Authentication required' });

  const user = await sessionStore.get(session);
  if (!user) return res.status(401).json({ error: 'Invalid session' });

  if (user.sessionExpiry < Date.now()) {
    await sessionStore.destroy(session);
    return res.status(401).json({ error: 'Session expired' });
  }

  req.user = user;
  next();
}

// Apply to protected routes
app.use('/api/protected', authenticate);
```

### Authorization Middleware

```javascript
function authorize(...roles) {
  return (req, res, next) => {
    if (!roles.includes(req.user.role)) {
      return res.status(403).json({ error: 'Insufficient permissions' });
    }
    next();
  };
}

app.delete('/api/users/:id', authenticate, authorize('admin'), deleteUser);
```

### Key Rules

- Authentication checks identity. Authorization checks permissions. Always separate them.
- Fail closed: deny by default, allow explicitly.
- Never leak whether a user exists in auth error messages ("Invalid credentials" not "User not found").
- Re-authenticate for sensitive operations (password change, email change, payment).

## Password Hashing

### Algorithm Selection

| Algorithm | Recommendation | Notes |
|---|---|---|
| **Argon2id** | Preferred | Memory-hard, resistant to GPU/ASIC attacks |
| **bcrypt** | Acceptable | Well-tested, widely supported, 72-byte input limit |
| **scrypt** | Acceptable | Memory-hard alternative |
| **PBKDF2** | Legacy only | Use only if above unavailable |
| MD5/SHA-*/SHA-*+salt | **Never** | Not suitable for password hashing |

### Argon2id Configuration

```javascript
import argon2 from 'argon2';

// Hash
const hash = await argon2.hash(password, {
  type: argon2.argon2id,
  memoryCost: 65536,    // 64 MB
  timeCost: 3,          // 3 iterations
  parallelism: 4,
});

// Verify
const valid = await argon2.verify(hash, inputPassword);
```

### bcrypt Configuration

```javascript
import bcrypt from 'bcrypt';

const SALT_ROUNDS = 12; // Minimum 10, prefer 12+

// Hash
const hash = await bcrypt.hash(password, SALT_ROUNDS);

// Verify
const valid = await bcrypt.compare(inputPassword, hash);
```

### Password Policy

- Minimum 8 characters (NIST SP 800-63B)
- No maximum under 64 characters
- Check against breached password lists (Have I Been Pwned API / k-anonymity model)
- Do not enforce complex composition rules (uppercase/symbol requirements) — they reduce entropy in practice
- Allow all Unicode characters and spaces

## Session Management

### Server-Side Sessions

```javascript
// Session configuration
const sessionConfig = {
  name: '__Host-session',
  secret: process.env.SESSION_SECRET,  // 256-bit minimum
  resave: false,
  saveUninitialized: false,
  cookie: {
    httpOnly: true,
    secure: true,           // HTTPS only
    sameSite: 'lax',
    maxAge: 3600000,        // 1 hour
    path: '/',
  },
  store: redisStore,        // Never use in-memory store in production
};
```

### Session ID Requirements

- Minimum 128-bit entropy (32 hex chars or 24 base64 chars)
- Generated by CSPRNG (`crypto.randomBytes`, not `Math.random`)
- Regenerate session ID after authentication (prevent session fixation)
- Invalidate on logout (server-side deletion, not just cookie removal)

### Session Lifecycle

```
Login --> Generate new session ID --> Set cookie
             |
Privilege change --> Regenerate session ID
             |
Logout --> Destroy session server-side --> Clear cookie
             |
Timeout --> Destroy session (idle: 30min, absolute: 8hr typical)
```

### Concurrent Session Control

```javascript
// Option: limit to N active sessions per user
async function enforceSessionLimit(userId, maxSessions = 3) {
  const sessions = await sessionStore.findByUser(userId);
  if (sessions.length >= maxSessions) {
    // Destroy oldest session
    await sessionStore.destroy(sessions[0].id);
  }
}
```

## Rate Limiting

### Strategy by Endpoint Type

| Endpoint | Window | Max Requests | Key |
|---|---|---|---|
| Login | 15 min | 5 failures | IP + username |
| Password reset | 1 hour | 3 | email |
| API (authenticated) | 1 min | 100 | user ID |
| API (unauthenticated) | 1 min | 20 | IP |
| Registration | 1 hour | 5 | IP |
| File upload | 1 hour | 10 | user ID |

### Implementation (Express)

```javascript
import rateLimit from 'express-rate-limit';
import RedisStore from 'rate-limit-redis';

const loginLimiter = rateLimit({
  store: new RedisStore({ client: redisClient }),
  windowMs: 15 * 60 * 1000,
  max: 5,
  message: { error: 'Too many attempts, try again later' },
  standardHeaders: true,       // RateLimit-* headers
  legacyHeaders: false,
  keyGenerator: (req) => `${req.ip}:${req.body?.username}`,
  skipSuccessfulRequests: true, // Only count failures
});

app.post('/auth/login', loginLimiter, loginHandler);
```

### Response Headers

```
RateLimit-Limit: 100
RateLimit-Remaining: 42
RateLimit-Reset: 1620000060
Retry-After: 30
```

Always return `429 Too Many Requests` with `Retry-After` header.

## Request Validation

### Schema Validation (Zod Example)

```javascript
import { z } from 'zod';

const CreateUserSchema = z.object({
  email: z.string().email().max(254),
  name: z.string().min(1).max(100).trim(),
  age: z.number().int().min(13).max(150).optional(),
});

function validateBody(schema) {
  return (req, res, next) => {
    const result = schema.safeParse(req.body);
    if (!result.success) {
      return res.status(400).json({
        error: 'Validation failed',
        details: result.error.issues,
      });
    }
    req.body = result.data; // Use parsed/sanitized data
    next();
  };
}

app.post('/users', validateBody(CreateUserSchema), createUser);
```

### Validation Rules

- Validate ALL input: body, query params, path params, headers
- Whitelist approach: define what is allowed, reject everything else
- Type coercion is a security concern — validate types strictly
- Reject unexpected fields (strip or error)
- Validate content-type header matches expected format
- Set `express.json({ limit: '100kb' })` to prevent large payload attacks

## SQL Injection Prevention

### Parameterized Queries (Always)

```javascript
// CORRECT — parameterized
const result = await db.query(
  'SELECT * FROM users WHERE email = $1 AND status = $2',
  [email, status]
);

// NEVER — string concatenation
const result = await db.query(
  `SELECT * FROM users WHERE email = '${email}'`  // VULNERABLE
);
```

### ORM Safety

```javascript
// Prisma — safe by default
const user = await prisma.user.findUnique({ where: { email } });

// Drizzle — safe by default
const user = await db.select().from(users).where(eq(users.email, email));

// Raw queries in ORMs — still parameterize
const result = await prisma.$queryRaw`SELECT * FROM users WHERE email = ${email}`;
// Prisma uses tagged template literals for parameterization

// DANGEROUS — Prisma raw with string interpolation
const result = await prisma.$queryRawUnsafe(`SELECT * FROM users WHERE email = '${email}'`);
```

### Dynamic Query Building

```javascript
// Safe dynamic WHERE clauses
function buildQuery(filters) {
  const conditions = [];
  const params = [];
  let paramIndex = 1;

  if (filters.status) {
    conditions.push(`status = $${paramIndex++}`);
    params.push(filters.status);
  }
  if (filters.role) {
    conditions.push(`role = $${paramIndex++}`);
    params.push(filters.role);
  }

  const where = conditions.length ? `WHERE ${conditions.join(' AND ')}` : '';
  return { query: `SELECT * FROM users ${where}`, params };
}
```

### Column/Table Name Safety

Column and table names cannot be parameterized. Whitelist them:

```javascript
const ALLOWED_SORT_COLUMNS = ['name', 'created_at', 'email'];
const sortCol = ALLOWED_SORT_COLUMNS.includes(input) ? input : 'created_at';
```

## File Upload Security

### Server-Side Validation

```javascript
import fileType from 'file-type';
import path from 'path';
import crypto from 'crypto';

const ALLOWED_MIME = ['image/jpeg', 'image/png', 'application/pdf'];
const MAX_SIZE = 10 * 1024 * 1024; // 10 MB

async function processUpload(file) {
  // 1. Check size
  if (file.size > MAX_SIZE) throw new Error('File too large');

  // 2. Detect actual MIME type from file bytes (not client header)
  const detected = await fileType.fromBuffer(file.buffer);
  if (!detected || !ALLOWED_MIME.includes(detected.mime)) {
    throw new Error('Invalid file type');
  }

  // 3. Generate random filename (never use original filename)
  const ext = path.extname(file.originalname).toLowerCase();
  const safeName = `${crypto.randomUUID()}${ext}`;

  // 4. Store outside web root or in object storage (S3, GCS)
  await objectStorage.upload(safeName, file.buffer, {
    contentType: detected.mime,
    contentDisposition: 'attachment', // Force download, prevent inline execution
  });

  return safeName;
}
```

### File Upload Rules

- Never serve uploaded files from the same domain as your app
- Use a separate domain or CDN with restricted CSP
- Strip EXIF/metadata from images (e.g., `sharp` library)
- Scan for malware if accepting documents (ClamAV integration)
- Set `Content-Disposition: attachment` on download endpoints
- Never execute or interpret uploaded file contents
- Limit upload frequency per user (rate limit)

## Server Hardening

### HTTP Security Headers

```javascript
// Helmet.js (Express) or manual configuration
app.use(helmet({
  contentSecurityPolicy: { /* see frontend-security.md */ },
  hsts: { maxAge: 31536000, includeSubDomains: true, preload: true },
  frameguard: { action: 'deny' },
  noSniff: true,
  referrerPolicy: { policy: 'strict-origin-when-cross-origin' },
  permittedCrossDomainPolicies: { permittedPolicies: 'none' },
}));

// Additional headers
app.use((req, res, next) => {
  res.setHeader('Permissions-Policy', 'camera=(), microphone=(), geolocation=()');
  res.removeHeader('X-Powered-By');
  next();
});
```

### Essential Headers

| Header | Value | Purpose |
|---|---|---|
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains; preload` | Force HTTPS |
| `X-Content-Type-Options` | `nosniff` | Prevent MIME sniffing |
| `X-Frame-Options` | `DENY` | Prevent clickjacking |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | Control referrer leakage |
| `Permissions-Policy` | `camera=(), microphone=()` | Restrict browser APIs |
| `Cache-Control` | `no-store` (for sensitive responses) | Prevent caching of PII |

### Process Security

- Run as non-root user
- Minimize installed packages in production image
- Disable debug modes, stack traces in production errors
- Set `NODE_ENV=production` (or equivalent)
- Limit file descriptors and memory allocation
- Use `--max-old-space-size` to cap Node.js memory

## Sensitive Data Logging Rules

### Never Log

- Passwords or password hashes
- Session tokens, JWTs, API keys
- Credit card numbers, CVVs
- Social Security Numbers, government IDs
- Full database connection strings with credentials
- Encryption keys or secrets
- Raw request bodies containing PII

### Safe Logging Pattern

```javascript
function sanitizeForLogging(data) {
  const sanitized = { ...data };
  const REDACT_FIELDS = [
    'password', 'token', 'secret', 'authorization',
    'cookie', 'ssn', 'creditCard', 'apiKey',
  ];

  for (const key of Object.keys(sanitized)) {
    if (REDACT_FIELDS.some(f => key.toLowerCase().includes(f))) {
      sanitized[key] = '[REDACTED]';
    }
  }
  return sanitized;
}

// Structured logging with redaction
logger.info('User action', {
  userId: user.id,
  action: 'login',
  ip: req.ip,
  // Never: password, token, session
});
```

### Error Responses

```javascript
// Production error handler — never expose internals
app.use((err, req, res, next) => {
  logger.error('Unhandled error', {
    error: err.message,
    stack: err.stack,    // Log stack trace server-side only
    requestId: req.id,
  });

  res.status(500).json({
    error: 'Internal server error',
    requestId: req.id,    // For support correlation
    // Never: stack trace, SQL error, internal paths
  });
});
```

## Quick Checklist

- [ ] Authentication middleware applied to all protected routes
- [ ] Passwords hashed with Argon2id or bcrypt (cost factor 12+)
- [ ] Session IDs: 128-bit entropy, regenerated on auth, server-side storage
- [ ] Rate limiting on login, registration, password reset, API endpoints
- [ ] All input validated with schema validation (Zod, Joi, etc.)
- [ ] All SQL uses parameterized queries — no string concatenation
- [ ] File uploads: MIME verified from bytes, random filenames, stored externally
- [ ] Security headers configured (HSTS, CSP, X-Frame-Options, etc.)
- [ ] Sensitive data never logged; error responses never expose internals
- [ ] Production mode enabled; debug output disabled
- [ ] Non-root process execution
- [ ] `X-Powered-By` and server version headers removed
